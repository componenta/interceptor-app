# Componenta Interceptor App

Интеграционный пакет для `componenta/interceptor` в Componenta-приложении. Он превращает атрибуты перехватчиков в сериализуемые описания, которые слой перехватчиков может загрузить из кеша.

Если коду нужно только выполнять цепочки перехватчиков, он должен зависеть от `componenta/interceptor`. Этот пакет нужен во время сборки кеша приложения.

## Установка

```bash
composer require componenta/interceptor-app
```

Пакет объявляет `Componenta\Interceptor\App\ConfigProvider` в `extra.componenta.config-providers`.
Если установлен `componenta/composer-plugin`, провайдер автоматически добавляется в сгенерированный список провайдеров.

## Связанные пакеты

| Пакет | Зачем нужен здесь |
|---|---|
| `componenta/interceptor` | Выполняет цепочки перехватчиков и использует скомпилированные описания. |
| `componenta/class-finder` | Находит классы и методы с атрибутами перехватчиков. |
| `componenta/app` | Подключает компилятор только когда `componenta/interceptor` установлен и связан в контейнере. |

## Что добавляет

Пакет предоставляет `Componenta\Interceptor\App\Compile\InterceptorMapCompiler`.

Компилятор сканирует найденные классы на:

- атрибуты `#[Intercept]` на методах;
- атрибуты, которые напрямую реализуют интерфейсы перехватчиков.

Он создаёт карту описаний, которую можно передать в `AttributeInterceptor` из `componenta/interceptor`.

## Режим разработки

В разработке метаданные перехватчиков могут читаться из классов во время сборки кеша приложения. Это сохраняет декларативное описание перехвата на методах и не заставляет рантайм-пакет зависеть от сканирования классов.

## Продакшен

В продакшене приложение должно загружать скомпилированную карту описаний. Это убирает повторную рефлексию по методам и делает подготовку перехвата предсказуемой.

Компилятор должен включаться только когда `componenta/interceptor` установлен и сервис атрибутного перехватчика связан в контейнере. Опциональное включение выполняется в `componenta/app`.

## Границы

`componenta/interceptor-app` не выполняет цепочки перехватчиков. Контексты вызова, выполнение цепочки, атрибуты перехватчиков, фабрики и отправка вызова во время выполнения принадлежат `componenta/interceptor`; этот пакет отвечает только за компиляцию описаний.
